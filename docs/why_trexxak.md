# Why Trexxak (v0.2)

Trexxak is positioned as an Intent Language for agent behavior scripting.

## Primary Market

AI-native agent behavior designers (indie builders and advanced creators) who need:
- compact symbolic authoring,
- inspectable execution traces,
- readable intent translation for review and collaboration.

## Competitive Map

### vs Prompt/Template DSLs

Trexxak advantage:
- explicit control-flow and symbol identity,
- direct executable runtime path.

Trexxak non-goal:
- prompt-only optimization without runtime semantics.

### vs Workflow Graph DSLs

Trexxak advantage:
- compact hand-authored scripts for fast behavior iteration,
- built-in human-readable translation.

Trexxak non-goal:
- enterprise orchestration concerns (durability, scheduling, infra-level retries).

### vs Typed Config DSLs

Trexxak advantage:
- behavior execution and narrative intent shaping.

Trexxak non-goal:
- static type theorem-style guarantees across large config systems.

## Decision Rule

Use Trexxak when your main problem is:
- expressing and reviewing agent behavior intent.

Use other stacks when your main problem is:
- production orchestration depth,
- strict schema/type proofs,
- infrastructure-scale workflow management.

## Strategic Positioning

Trexxak is not sold as "faster prototyping syntax".
Trexxak is presented as a disciplined Intent Language with:
- strict semantics,
- conformance evidence,
- deterministic runtime traces.
